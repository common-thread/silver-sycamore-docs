<task type="auto">
  <name>Task 3: Create transposition methodology</name>
  <files>.planning/phases/26-transposition-process-design/METHODOLOGY.md</files>
  <action>
Create METHODOLOGY.md with step-by-step process for transposing documents. This is the playbook for Phase 27 execution.

**Methodology sections:**

1. **Overview**
   - 62 documents total (from AUDIT.md)
   - Batch-by-type approach (all procedures together, etc.)
   - Estimated effort by priority tier

2. **Pre-Transposition Checklist**
   - Verify document classification correct
   - Check for broken images/links
   - Note any special handling needs

3. **Transposition Process (per document)**
   Step 1: Read original document, identify intent
   Step 2: Check classification (apply reclassification if flagged)
   Step 3: Add/update frontmatter (title, description, contentType)
   Step 4: Apply template structure
   Step 5: Fix rendering issues (whitespace, broken refs)
   Step 6: Verify formatting against template

4. **Batch Execution Order**
   - Priority 1: Quick wins (28 docs) — frontmatter + minor fixes
   - Priority 2: Medium effort (24 docs) — restructuring
   - Priority 3: Heavy lift (10 docs) — rewrites + decisions

5. **Quality Checks**
   - Frontmatter validation
   - Content type compliance
   - No broken links/images
   - Intent preserved

6. **Edge Case Handling**
   - When to use fallback template
   - When to escalate to user
   - Large files (>25KB) handling

7. **Completion Criteria**
   - What "done" looks like for each document
   - Phase 27 handoff checklist

**Reference:**
- TEMPLATES.md (created in Task 2) for specific formats
- AUDIT.md Section 8 for prioritized list
- CONTEXT.md for escalation approach (ask user when unsure)

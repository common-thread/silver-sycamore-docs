# Plan 06-02: Personal Workspace UI

## Objective

Build the frontend interface for users to view, create, and organize their personal documents and folders.

## Context

- Personal documents/folders backend exists (Plan 06-01)
- Design system established (Phase 1)
- Users have authentication and profiles
- Need `/workspace` route for personal area

## Tasks

### 1. Create Workspace Layout
**File**: `src/app/workspace/layout.tsx`

- Sidebar with folder tree
- Main content area for document list/editor
- "New Document" and "New Folder" actions
- Uses ContentBox and design tokens

### 2. Create Workspace Page
**File**: `src/app/workspace/page.tsx`

- Shows documents at root level (no folder)
- Folder navigation (breadcrumbs + list)
- Empty state for new users
- Document cards with title, updated date

### 3. Create Folder View Component
**File**: `src/components/FolderTree.tsx`

- Collapsible folder hierarchy
- Click to navigate into folder
- Visual indicator for current folder
- Create new folder inline

### 4. Create Personal Document List
**File**: `src/components/PersonalDocList.tsx`

- Grid or list of documents in current folder
- Shows title, last updated, source indicator (if copied)
- Click to open document
- Quick actions: rename, move, delete

### 5. Create Personal Document Page
**File**: `src/app/workspace/[docId]/page.tsx`

- View/edit personal document
- Auto-save on changes (debounced)
- Back to workspace navigation
- Delete confirmation

### 6. Create Document Editor
**File**: `src/components/PersonalDocEditor.tsx`

- Title input field
- Markdown content textarea
- Save status indicator
- Preview toggle (optional)

### 7. Add Workspace Link to Header
**File**: `src/components/Header.tsx`

- Add "My Workspace" link for authenticated users
- Icon + text, links to `/workspace`

## Verification

- [ ] `/workspace` shows empty state for new users
- [ ] Can create folder from UI
- [ ] Can create document in folder
- [ ] Folder navigation works
- [ ] Document editor saves changes
- [ ] TypeScript compiles

## Design Notes

- Match existing design tokens
- Folder icons use simple outline style
- Document cards match CategoryGrid aesthetic
- Editor uses monospace for content (markdown editing)
